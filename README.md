# Editorial Design Knowledge DB (EDD)

紙面デザイン（書籍・雑誌・レポート・新聞・カタログ等）の世界中のベストプラクティスを構造化したナレッジベース。

## スコープ
- **学習ソース**: 多言語（英・仏・独・中・韓・伊等）
- **最終アウトプット**: 日本語制作支援
- **対象媒体**: 書籍・雑誌・レポート・新聞・カタログ・ZINE・パンフレット
- **目標規模**: 約20,000レコード（うちデザインパターン10,000+）

## レイヤー構造
- L0: 理論基盤（タイポグラフィ理論・知覚論等）
- L1: フォント知識（和文500+、欧文300+、書体メトリクス）
- L2: 座標構造（判型・グリッド・版面率）
- L3: デザインパターン（10,000+）
- L4: 巨匠・スタジオ・流派
- L5: コピー・タイトル技法
- L6: 作例コーパス

## ディレクトリ
- `research/` — 6トラック基礎リサーチ成果
- `schema/` — DBスキーマ定義
- `data/` — SQLite DB本体・CSV
- `scripts/` — 構築・検証スクリプト
- `reports/` — 検証レポート・分析

## 関連DB
- Writing Craft DB（文章技法・相互参照）
- Great Figures DB（人物系譜・相互参照）
- DDB（Webデザイン・補完関係）

## 構築フェーズ
- Phase 1: 基礎リサーチ（6トラック並列） ← 現在
- Phase 2: 分類体系統合・スキーマ確定
- Phase 3: academic-db-build + codex 20+並列収集
- Phase 4: 検証・正規化・エージェント化（/edd）
