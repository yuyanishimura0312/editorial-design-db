# Phase 3: 大規模収集計画（codex 20+並列）

## 目標規模
合計 **約20,000レコード**（うちデザインパターン10,000+）

## 20+ codex並列割り当て計画

### A. フォント収集チーム（5体）

- **A3**: 欧文古典セリフ（Garamond/Caslon/Bodoni系列の派生・地域版）→ 200書体
- **A4**: 欧文サンセリフ・現代書体（Helvetica/Inter/Söhne/GT系列）→ 200書体
- **A5**: 書体デザイナー（和欧合計200名）+ペアリング推奨50組

### B. 巨匠・スタジオチーム（4体）
- **B1**: 日本書籍デザイナー（杉浦康平・平野甲賀・鈴木一誌・祖父江慎・川名潤等80人＋代表作各5冊）

- **B3**: 海外雑誌AD（Brodovitch/Wolf/Fleckhaus/Brody/Carson等70人＋伝説的特集）
- **B4**: 日本雑誌AD（堀内誠一・仲條正義・木村裕治等50人＋代表特集）+出版社・スタジオ100組織

- **C3**: 雑誌パターン1500件（モード/カルチャー/ライフスタイル/ビジネス/思想ジャンル別）
- **C4**: 雑誌パターン1500件（表紙・目次・特集扉・記事フォーマット・コラム）
- **C5**: レポート・年次報告書パターン1000件

## 並列実行プロトコル
1. 各codexは担当領域のみインサート、ID衝突回避のためテーブル別に並列
2. 共有スキーマ参照（schema_v0_draft.sql）
3. 各codex終了時に自テーブルレコード数をログ出力
4. 関係テーブル（pattern_designer_links等）は全体収集後の後処理
